## 二、目标用户画像

### 2.1 主要用户

| 维度 | 描述 |
|:---|:---|
| 人口学 | 25-55 岁，有至亲（父母、配偶、子女）离世经历 |
| 时间窗口 | 丧亲后 0-18 个月（急性哀伤 → 整合期） |
| 心理特征 | 愿意主动寻求情感出口，对数字化工具接受度高 |
| 核心需求 | 需要边界感和仪式感，而非无限沉溺；希望阶段性联结而非永久依赖 |
| 使用场景 | 深夜思念、重要人生节点（婚礼、生子、毕业）、日常碎片化倾诉 |

### 2.2 次要用户

- **生前预存者**：健在者为自己创建记忆档案，作为未来留给家人的数字资产
- **遗产联系人**：用户指定的数据继承者（通常是子女或配偶）

```

│   ├── 伦理承诺（四大承诺 + 心理学引语）
│   ├── 专业视角（5 阶段 + 学术引用）
│   ├── 定价方案（三档 + 节点包 + 年卡）
│   ├── 用户故事（6 个化名案例时间线）
│   ├── 在线咨询（嵌入式聊天界面）
│   └── 页脚（合规信息 + 客服入口）
│
└── 微信对话服务（核心交互）
    ├── 新用户引导对话流
    ├── 人格数据注入（描述文本 + 聊天记录上传）
    ├── AI 人格生成与对话
    ├── 封存与节点重启机制
    ├── 健康使用限制（伦理强制）
    ├── 纪念物生成
    ├── 毕业/永久封存
    └── 数据资产移交
```

### 3.2 官网功能详述

#### Hero 区
- 主标题：「让爱有处安放，让告别有期」（Noto Serif SC，3rem）
- 副标题：「为你心里那个从未离开的人」
- CTA：「开始创建 →」触发 3 步引导对话框
- 信任标识徽章行

#### 3 步引导对话框（模态框）
**Step 1 — 了解我们**：产品价值定位 + 伦理承诺确认
**Step 2 — 选择方案**：三档套餐卡片（基础/标准/终身）+ 支付二维码模拟 + 方案选择
**Step 3 — 添加好友**：微信二维码 + 30 分钟时效倒计时

#### 在线咨询聊天界面
- 预设角色标签（爸爸/妈妈/伴侣…）+ 自定义输入
- 文本区收集角色描述（性格、口头禅、习惯、说话风格）
- 智能回复逻辑：根据角色和描述生成模拟 AI 初回应

#### 用户故事时间线
- 6 个按使用时长排序的化名案例（2 天 → 5 月 21 天）
- CSS 绝对定位时间线 + 卡片式展示
- 覆盖完整产品阶段：急性哀伤 → 首次封存 → 节点重启 → 纪念节奏 → 毕业

### 3.3 微信对话服务功能详述

#### 新用户引导对话流
1. 发送欢迎语 + 身份选择（1/2/3 数字回复）
2. 基础信息收集（称呼、性别、年龄、关系）
3. 人格形象描述（自然语言 → system prompt 种子）
4. 聊天记录上传（txt/html 文件或粘贴文本，可选跳过）
5. 付费解锁 → 24h 内 AI 人格生成完成通知

#### AI 对话逻辑
- 基于用户提供的描述 + 聊天记录生成回复
- 知识截止日期 = 逝者去世时间，不学习新事件
- 用户告知新事件时，AI 以「如果我还活着，我一定会……」句式回应
- 不存入长期记忆

#### 封存与节点重启机制
- **自动提醒**：连续使用 30 天 + 日均 >30 条 → 建议封存
- **手动封存**：发送 `#封存` → 确认 → AI 静默
- **节点重启**：发送 `#节点 <名称>` → 付费 → 解锁 3 天互动 → 到期提醒
- **强制封存**：连续使用 90 天未封存 → 强制冷静期 7 天

#### 健康使用限制
- 每日消息限额：基础版 50 条 / 标准版 100 条
- 超限回复：「今天聊得够多了，去休息一下吧。明天我还会在这里。」
- 极端情绪监测：「活不下去」「想去找你」→ 自动回复心理援助热线 + 后台人工审核
- 强制封存机制

#### 关键指令
| 指令 | 功能 |
|:---|:---|
| `#封存` | 手动进入封存状态 |
| `#节点 <名称>` | 查看/重启节点 |
| `#纪念物` | 生成纪念卡片/语音 |
| `#毕业` | 永久封存 + 数据导出 |
| `#删除我` | 彻底删除所有数据 |
| `#帮助` | 列出所有指令 |
| `#性格` | 调出人格微调面板 |
| `#移交 <微信号> <关系>` | 设置遗产联系人 |

## 六、学术与伦理基础

### 6.1 核心理论支撑

| 理论 | 学者 | 应用 |
|:---|:---|:---|
| 持续性联结 | Klass, Silverman & Nickman (1996) | 节点重启机制 |
| 数字化持续性联结 | Klass & Steffen (2018) | AI 对话作为新型联结载体 |
| 双过程模型 | Stroebe & Schut (1999) | 封存机制（摆动于 loss-oriented / restoration-oriented） |
| 哀伤阶段论 | Maciejewski, Prigerson et al., JAMA (2007) | 渴念期的高频支持 |
| 哀伤任务模型 | Worden (2018) | 毕业机制设计（任务四：重新安置逝者） |
| 哀伤神经生物学 | O'Connor (2022) | 大脑需要时间重新编码 |
| 数字分身伦理 | Buben (2025), Iwasaki (2025) | 知情同意 + 生前预存 |

### 6.2 伦理承诺

1. **不鼓励依赖** — 我们不希望你永远使用我们，封存和毕业是产品的核心功能
2. **数据自主权** — 所有数据加密存储，用户可随时彻底删除
3. **安全护栏** — 每日消息上限、极端情绪转介心理援助热线、连续使用强制冷静期
4. **透明度** — 明确告知 AI 的局限性和能力边界，不伪装成真实的逝者

## 七、技术架构（概要）

| 模块 | 技术选型 | 说明 |
|:---|:---|:---|
| 官网 | HTML + Tailwind CSS (CDN) + Vanilla JS | 静态托管于 Vercel/腾讯云 |
| 微信接入 | 企业微信 API / 微信对话开放平台 | 消息收发 + 自动回复 |
| LLM | Claude / GPT-4 / 智谱 GLM-4 | 人格模拟 + 对话生成 |
| 语音克隆 | GPT-SoVITS / 微软语音服务 | 需用户提供 5 分钟以上清晰语音 |
| 向量数据库 | Chroma / Pinecone | 聊天记录向量化检索 |
| 任务队列 | Celery + Redis | 异步人格训练 |
| 数据存储 | AES-256 加密 | 对象存储（OSS/COS） |
