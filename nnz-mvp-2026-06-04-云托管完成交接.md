### 遇到的坑与修复

**坑 1：Render 首次部署失败。**

错误：`npm error Missing script: "start"`

根因：本地 `package.json` 在 commit `f92699b` 中加了 `start` 脚本，但当时还没推送。Render 拉的是旧 commit `ae8633a`。

修复：用 Personal Access Token 命令行推送（`git push origin main`），Render 自动重新部署成功。

**坑 2：`dist-cjs/package.json` 缺失。**（已在上一次会话修复，今天确认生效）

根因：`.gitignore` 排除 `dist-cjs/`，tsc 编译不自动创建 `package.json`。根 `package.json` 的 `"type": "module"` 会让 Node 把 CommonJS 产物当 ESM 处理。

修复：`build:demo` 脚本末尾加了自动写入 `{"type":"commonjs"}` 的步骤。Render 上 `npm run build:demo` 会自动执行。

**坑 3：Codex 沙箱 Git 无凭据。**

普通 `git push` 会报 `could not read Username`。

解决方案：用户浏览器生成 GitHub Personal Access Token（repo 权限），然后 `git remote set-url origin https://TOKEN@github.com/...`。后续 push 都可用这个方式。
