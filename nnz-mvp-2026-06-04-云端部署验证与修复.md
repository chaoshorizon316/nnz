### 发现并修复的问题

**`dist-cjs/package.json` 缺失。**

根因：`.gitignore` 排除了 `dist-cjs/`，且 `tsc` 编译只产出 `.js` 文件，不会自动生成 `package.json`。根 `package.json` 的 `"type": "module"` 会让 Node 把 `dist-cjs/*.js` 当 ESM 处理，导致 `exports is not defined` 错误。

修复：`package.json` 的 `build:demo` 脚本改为：

```json
"build:demo": "tsc -p tsconfig.demo.json && node -e \"require('fs').writeFileSync('dist-cjs/package.json', JSON.stringify({type:'commonjs'})+'\\n')\""
```

编译后自动创建 `dist-cjs/package.json`，内容 `{"type":"commonjs"}`。

**验证**：`npm run build:demo` 后会生成 `dist-cjs/package.json`，`npm start` 可以成功启动构建产物。

## Render 构建流程（修复后）

```
git push → Render 检测到 main 更新
  ↓
npm ci                    ← 安装依赖
  ↓
npm run build:demo         ← tsc 编译 + 自动创建 dist-cjs/package.json
  ↓
npm run start              ← node dist-cjs/demo-server.js（CommonJS 模式正常加载）
  ↓
监听 0.0.0.0:$PORT
```
