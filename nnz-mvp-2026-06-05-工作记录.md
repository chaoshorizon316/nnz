## 晚间补充：Prompt 重构 — A/B 回复差异

### 问题

用户发现 A 和 B 的 LLM 回复趋同。根因分析后发现 `generateLlmReply` 有三个设计缺陷：

1. **无对话历史** — LLM 每次只看到孤立的单条消息，不感知上下文
2. **人格描述扁平** — `humorLevel=high` 只是一个词，缺少行为展开
3. **关系只是字符串** — "女儿心中的父亲" 没有转化成具体的说话风格指引
4. **口头禅未展开** — 只取了第一个 petPhrase（`readFirstString`），丢了列表中的其余口头禅

### 修复

Prompt 重构为五层分化：

| 层级 | 内容 | 效果 |
|:---|:---|:---|
| [WHO YOU ARE] | 该用户的 DESCRIPTION + CORRECTION 记忆全文 | A: "爸爸其实很幽默，只是不太主动开玩笑" vs B: 无此记忆 → 不同的幽默表达 |
| [关系风格] | 根据 relationship 生成行为指引 | A（女儿）→ tender concern, protective; B（儿子）→ steady encouragement, independence |
| [对话历史] | 最近 6 轮对话作为 user prompt 前缀 | A 聊婚礼 → 回复偏温情; B 聊工作 → 回复偏鼓励 |
| [幽默度] | 展开为行为描述 | high→"often cracks jokes, respects boundaries"; low→"speaks plainly, shows care through actions" |
| [口头禅] | 完整列表，要求自然融入 | A: "你自己拿主意"; B: "慢慢来" |

### 相关代码变更

- `generateLlmReply` 新增第 4 个参数 `recentConversations`
- 新增 `readStringArray` 辅助函数（读取完整口头禅列表）
- `sendMessageToBothUsers` 的调用处传入 `fixture.store.listConversations(scope)`
