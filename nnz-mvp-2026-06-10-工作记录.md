# nnz-mvp 2026-06-10 工作记录与交接

> 今日目标：先不改微信客户端，把真实用户验证流程切到首页 H5。首页不再嵌入 `/demo` 的 A/B 内部页，而是直接验证“注册/登录 -> 创建记忆中的人 -> 私密聊天”的用户端闭环。

## 当前状态

已完成：

- `/` 首页在线体验区改为 H5 用户端入口。
- `/demo` 仍保留为开发者验证页，用于 A/B 双用户隔离、Soul Ops、Proposal 等内部检查。
- 新增 auth-aware 用户端 API：
  - `GET /api/me`
  - `GET /api/me/personas`
  - `POST /api/me/persona`
  - `GET /api/me/chat-history?personaId=...`
  - `POST /api/me/chat`
- 用户端 API 使用 token 内的 `userId`，不接受前端传 `userId`。
- 用户端 API 仍需要 `personaId` 定位对话，但跨用户访问会返回温和 403：`没有权限访问这段对话。`
- 用户端 API 不返回 `SoulVersion`、`SoulSnapshot`、`MemoryItem`、`SoulUpdateProposal`、`kernelJson`、状态机英文名等内部机制字段。

## 已知边界

- 首页 H5 已能替代微信客户端先验证真实用户私有 Soul 链路。
- 这还不是完整微信产品；微信后续应复用 `/api/me/*` 的 auth-aware 设计。
- Render 免费 Web Service 如果不接 Postgres，重启后数据仍可能丢失。
- 当前用户端创建 persona 时还比较轻量：称呼、关系、口头禅、描述。后续可扩展上传资料、更多记忆采集、节点和封存入口。
